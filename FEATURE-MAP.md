# LeadRipper Feature Map

LeadRipper should be structured as a clear outbound operating system, not a pile of disconnected power-user tools.

## 1. Lead Engine
This is the wedge and the most differentiated part of the product.

### Core features
- Google Places lead scraping
- City + niche targeting
- Lead lists
- Contact enrichment
- Email discovery
- Email validation
- Website scoring
- Website rebuild trigger
- Lead tagging
- Search + filtering

### Why it matters
This is what makes LeadRipper feel different from standard cold email tools. It starts upstream, where agencies actually source opportunity.

## 2. Outreach Engine
This is the ManyReach / Instantly / Smartlead comparison surface.

### Core features
- Unlimited sender accounts
- Unlimited warmup
- Sender rotation
- Smart inbox rotation
- Campaign composer
- Multi-step sequences
- A/B testing
- Dynamic personalization / merge tags
- Spintax
- Spam checker
- Bounce handling
- Real-time analytics
- Unified inbox
- Send scheduling
- Template library

### Why it matters
This is the execution layer that turns scraped and enriched leads into actual conversations.

## 3. Sales OS / CRM
This is where LeadRipper becomes bigger than a scraping or cold email tool.

### Core features
- Built-in CRM
- Pipeline stages
- Lead/deal status
- Notes
- Tasks / reminders
- Activity timeline
- Team seats
- Client workspaces
- Account ownership / assignment

### Why it matters
Agencies need a place to work the deal after outreach. Otherwise LeadRipper feels like a data extractor with nowhere for the opportunity to live.

## 4. Calling + Comms Layer
This is the bridge between outbound and direct conversations.

### Core features
- Twilio integration
- Softphone
- Click-to-call
- Call status / logging
- ElevenLabs integration
- AI calling / voice agents
- Call result logging
- No-email lead fallback calling flow

### Why it matters
This makes LeadRipper useful for leads that do not have email, and lets agencies run a true multichannel outbound workflow.

## 5. Integrations Layer
This is what helps the product fit into an agency's existing operations.

### Core features
- GoHighLevel sync
- API access
- Webhooks
- Export / import
- Future: Zapier / Make / Slack

### Why it matters
This reduces switching costs and makes LeadRipper easier to adopt without forcing every team to abandon their current stack.

## 6. Enablement Layer
This is what makes the product usable by normal humans.

### Core features
- Setup guides
- Integration tutorials
- Tooltips
- Quick start onboarding
- Deliverability tips
- Best-practice checklists
- Help page / setup hub

### Why it matters
Without this, a powerful product feels overwhelming and support-heavy.

---

# Recommended top-level product structure

Use navigation that reflects how agencies actually work:

- Leads
- Campaigns
- Inbox
- Pipeline
- Calling
- Integrations
- Settings
- Help

# Positioning summary

LeadRipper is best framed as:

**An agency outbound operating system**

Not just:
- a cold email tool
- a lead scraper
- a CRM

But a system that helps agencies:
1. source opportunity
2. enrich and qualify it
3. launch outreach
4. call prospects
5. manage the pipeline
6. sync important data into external systems
